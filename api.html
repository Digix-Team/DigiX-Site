ایجاد درخواست

تمامی درخواست‌های مربوط به وب سرویس سایت digix-helper باید از طریق HTTPS و به شکل زیر ارسال شوند:

https://api.digix-helper.ir/METHOD_NAME?token=your_token

مثال درخواست:

GET https://api.digix-helper.ir/users?token=123456ABCDEF

متدهای پشتیبانی شده

تمامی وب سرویسها متدهای GET و POST را پشتیبانی می‌کنند.

روش‌های ارسال پارامتر

  • URL query string
  • application/x-www-form-urlencoded
  • application/json
  • multipart/form-data (برای آپلود فایل)

فرمت پاسخ

پاسخ همیشه یک شیء JSON است که شامل فیلد ok از نوع boolean است. ممکن است فیلد description برای توضیح نتیجه درخواست وجود داشته باشد. اگر ok برابر true باشد، درخواست موفق بوده و نتیجه در فیلد result قرار دارد. در صورت عدم موفقیت، ok برابر false خواهد بود و خطا در description توضیح داده می‌شود. همه پاسخ‌ها ممکن است شامل error_code (integer) و در برخی موارد فیلد اختیاری parameters از نوع ResponseParameters باشند.

نکات مهم

  • تمامی درخواست‌ها باید شامل پارامتر token باشند.
  • هیچ حساسیتی نسبت به بزرگ یا کوچک بودن حروف وجود ندارد.
  • تمامی درخواست‌ها باید با فرمت UTF-8 ارسال شوند.
  • برای ارسال پارامترها از یکی از Content-Typeهای application/json، application/x-www-form-urlencoded یا multipart/form-data استفاده کنید.
  • متد درخواست خود را از طریق پارامتر method مشخص کنید.
